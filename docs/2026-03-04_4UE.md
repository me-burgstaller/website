## Voucher Controller
Nun habe ich die Grundlage geschaffen, dass im System Gutscheine angelegt werden können. Ein Gutschein soll immer einem bestimmten Schüler oder einer bestimmten Schülerin zugeordnet sein und zuerst erstellt werden. Danach kann er später noch genehmigt, versendet und schließlich eingelöst werden. Ein Controller übernimmt in Laravel die Logik zwischen Datenbank und Benutzeroberfläche.

Der Controller wird später dafür verwendet:
- Gutscheine anzuzeigen
- neue Gutscheine zu erstellen
- Gutscheine zu speichern

Damit dieser Ablauf möglich ist, musste ich zuerst die passenden Models, Controller und Routen erstellen.
Ich habe mit `php artisan make:controller VoucherController` meinen ersten Controller angelegt. 
Dann habe ich standardmäßig mal 3 Funktionen erstellt: `index`, `create` und `store`.

### Index
In der `index`-Funktion habe ich als erstes alle vorhanden Gutscheine aus der Datenbank geladen. Dabei habe ich zusätzlich die Beziehung zum jeweiligen Schüler geladen `with('student')`, damit später in der Übersicht auch direkt der Name vom Schüler angezeigt werden kann. Ich habe die Gutscheine auch nach dem neuesten Gutschein sortiert, die aktuellsten Gutscheine sind oben ersichtlich. Dann werden die Daten an die View `vouchers.index` weitergegeben, damit sie dort in einer Tabelle dargestellt werden können.

### Neue Route
Dafür habe ich jedoch erstmals die `vouchers.index` in `web.php` hinzufügen müssen. Dabei habe ich diese passenden Routes erstellt:
```
Route::middleware(['auth'])->group(function () {
    Route::get('/vouchers', [VoucherController::class, 'index'])->name('vouchers.index');
    Route::get('/vouchers/create', [VoucherController::class, 'create'])->name('vouchers.create');
    Route::post('/vouchers', [VoucherController::class, 'store'])->name('vouchers.store');
});
```
Alle diese Routes wurden zusätzlich mit der auth-Middleware geschützt. Das bedeutet, dass nur eingeloggte Benutzer/innen auf diese Funktionen zugreifen können. Das ist wichtig, weil Gutscheine im System nur von autorisierten Personen erstellt werden sollen und nicht öffentlich zugänglich sein dürfen.

### Create
Die `create`-Funktion dient dazu, das Formular zum Erstellen eines neuen Gutscheins anzuzeigen. Dafür habe ich zuerst alle Schüler/innen aus der Datenbank geladen und alphabetisch nach Nachname sortiert. Diese werden später in einem Dropdown-Menü angezeigt damit beim Erstellen eines Gutscheins die richtige Person ausgewählt werden kann. Zusätzlich habe ich die möglichen Gutscheinvorlagen (excellent_success, good_success, event_help, custom), sowie die möglichen Gutscheinarten (day, afternoon, hour) als `arrays` definiert. Dadurch kann ich diese Werte einfach im Formular anzeigen und gleichzeitig sicherstellen, dass nur gültige Optionen ausgewählt werden können.

### Store
Die `store`-Funktion ist dafür zuständig, einen neuen Gutschein in der Datenbank zu speichern. Zuerst werden alle Eingaben validiert, damit keine falschen oder unvollständigen Daten gespeichert werden. Dabei wird zum Beispiel überprüft, ob ein gültiger Schüler ausgewählt wurde, ob die Gutscheinart korrekt ist und ob eine gültige Anzahl angegeben wurde. Wenn im Formular die Vorlage „custom“ ausgewählt wird, muss zusätzlich ein individueller Grund (`reason_custom`) eingegeben werden.

Wenn alle Daten gültig sind, wird ein neuer Gutschein erstellt und in der Datenbank gespeichert. Dabei wird automatisch die aktuell eingeloggte Person als Aussteller (`issued_by_user_id`) gesetzt. Außerdem wird der Zeitpunkt der Erstellung (`issued_at`) mit `now()` gespeichert. Der Status des Gutscheins wird zunächst auf `pending` gesetzt, da der Gutschein später noch genehmigt werden muss.

Zum Schluss wird der Benutzer wieder zur Gutscheinübersicht weitergeleitet und erhält eine kurze Bestätigung, dass der Gutschein erfolgreich erstellt wurde.

## Aufbau des Formulars in vouchers.create
In der View `vouchers.create` habe ich anschließend das eigentliche Formular zum Erstellen eines Gutscheins umgesetzt. Das Formular sendet seine Daten per `POST` an die Route `vouchers.store`, welche im `VoucherController` verarbeitet wird. Zusätzlich wird das Laravel-Sicherheitsfeature `@csrf` verwendet.

Zu Beginn des Formulars wird überprüft, ob es Validierungsfehler gibt. Falls bei der Eingabe Fehler auftreten (z. B. wenn ein Pflichtfeld nicht ausgefüllt wurde), werden diese oberhalb des Formulars in einer Liste angezeigt. Dadurch erhält der Benutzer direkt eine Rückmeldung, welche Eingaben korrigiert werden müssen.

Im ersten Schritt kann eine Schülerin oder ein Schüler aus der Datenbank ausgewählt werden. Dazu wird ein Dropdown-Menü verwendet. Im Dropdown wird der Nachname, der Vorname sowie die Matrikelnummer angezeigt, damit die richtige Person eindeutig ausgewählt werden kann.

Danach kann die Gutscheinvorlage ausgewählt werden. Die möglichen Vorlagen werden im Controller als Array definiert und anschließend im Formular angezeigt. Ein besonderes Feld ist der individuelle Grund (reason_custom). Dieses Feld wird nur benötigt, wenn im Dropdown-Menü die Vorlage „Individuell“ ausgewählt wird. Damit das Formular übersichtlich bleibt, wird dieses Feld standardmäßig ausgeblendet. Mithilfe eines kleinen JavaScript-Skripts wird überprüft, welche Vorlage ausgewählt wurde. Sobald die Option `custom` ausgewählt wird, erscheint das Eingabefeld automatisch. Wenn eine andere Vorlage ausgewählt wird, wird das Feld wieder ausgeblendet.

Anschließend kann die Art des Gutscheins ausgewählt werden. Dabei stehen die Optionen Tag, Nachmittag oder Stunde zur Verfügung. Zusätzlich muss die Anzahl der Tage, Nachmittage oder Stunden eingegeben werden.

Optional kann außerdem ein Ablaufdatum für den Gutschein festgelegt werden. Wenn kein Datum angegeben wird, bleibt dieses Feld in der Datenbank leer. Zum Abschluss enthält das Formular einen Button, mit dem der Gutschein gespeichert werden kann. Beim Absenden des Formulars werden die Daten an die `store-Methode` im `VoucherController` übergeben und dort weiterverarbeitet.

## Darstellung der Gutscheine in vouchers.index
In der View vouchers.index habe ich eine Tabelle erstellt, in der alle vorhandenen Gutscheine angezeigt werden. Zunächst wird überprüft, ob bereits Gutscheine vorhanden sind. Falls keine Gutscheine existieren, wird eine kurze Nachricht angezeigt, dass noch keine Gutscheine im System gespeichert sind.

Wenn Gutscheine vorhanden sind, werden diese in einer Tabelle dargestellt. Die Tabelle enthält mehrere Spalten, darunter sind die ganzen Daten:
- Gutscheincode
- Schüler/in
- Gutscheinvorlage
- Art (Tag,...)
- Anzahl 
- Status
- Zeitpunkt der Ausstellung
- Ablaufdatum

Zusätzlich wird oberhalb der Tabelle ein Link angezeigt, mit dem ein neuer Gutschein erstellt werden kann. Dieser führt direkt zur Seite `/vouchers/create`. Am Ende der Tabelle wird außerdem die Laravel-Funktion `paginate()` verwendet, um die Gutscheine seitenweise darzustellen.
