## Schüler Dropdown Auswahl
Am Anfang habe ich bei der Erstellung eines Gutscheins ein klassisches Dropdown-Menü verwendet, um Schüler auszuwählen. Es wurden alle Schüler/Innen aus der Datenbank geladen und angezeigt. Es hat funktioniert, aber es war unübersichtlich. Deshalb habe ich nun eine Suchfunktion dazuprogrammiert.

### Schülersuche (Autocomplete)
Beim Feld „Schüler:in suchen“ kann man nun Anfangsbuchstabe(n) des Vor- oder Nachnamens eingeben und es kommen passende Ergebnisse. Ebenso kann man anfangen, die Matrikelnummer einzugeben und es erscheinen passende Schüler/innen mit Nummern. Die gewünschte Person kann anschließend direkt angeklickt werden. Weiters kann man auch die Matrikelnummer komplett eingeben und dann ENTER drücken, wodurch der Schüler bzw. die Schülerin direkt ausgewählt wird. Genau gleich funktioniert es auch beim Namen. Dabei ist die Klein- und Großschreibung egal, ebenso die Reihenfolge, also Vorname Nachname oder Nachname Vorname.

Nach der Auswahl wird die Person im Formular angezeigt und intern wird die entsprechende `student_id` gespeichert, damit beim Erstellen des Gutscheins eindeutig der richtige Schüler bzw. die richtige Schülerin verwendet wird.

### Umsetzung der Suche
Für die Suche wurde im `StudentController` eine zusätzliche Methode `search()` erstellt. Diese Methode verarbeitet die Suchanfrage und liefert passende Schüler/innen aus der Datenbank zurück.

```php
public function search(Request $request)
    {
        $query = trim($request->get('q', ''));

        if ($query === '') {
            return response()->json([]);
        }

        $students = Student::query()
            ->where(function ($q) use ($query) {
                $q->where('first_name', 'like', $query.'%')
                    ->orWhere('last_name', 'like', $query.'%')
                    ->orWhere('student_id', 'like', $query.'%')
                    ->orWhereRaw("CONCAT(first_name, ' ', last_name) like ?", [$query.'%'])
                    ->orWhereRaw("CONCAT(last_name, ' ', first_name) like ?", [$query.'%']);
            })
            ->orderBy('last_name')
            ->orderBy('first_name')
            ->limit(10)
            ->get([
                'id',
                'student_id',
                'first_name',
                'last_name',
            ]);

        return response()->json($students);
    }
```
Ebenso kann man die komplette Matrikelnummer eingeben und mit ENTER bestätigen und der Schüler wird ausgewählt. Genau gleich funktioniert es auch beim Namen, da ist die Klein- und Großschreibung, sowie Reihenfolge (Vorname dann Nachname oder andersherum) egal.

## Schülererstellung beim Gutschein erstellen
Es kann vorkommen, dass es einen Schüler im System noch nicht gibt und das merkt man erst bei der Gutscheinerstellung selbst. Deswegen habe ich es so gelöst, dass unter den Suchergebnissen ein Hinweis erscheint, dass keine passende Person gefunden wurde. Zusätzlich wird dann ein Link angezeigt, über den die Person einen neuen Schüler erstellen kann. 

Beim Draufklicken des Links, wird die zuvor eingegebene Matrikelnummer automatisch übernommen. Voraussetzung ist das nur Zahlen eingegeben wurden - dadurch muss man die Nummer nicht erneut eingeben. Ich wollte das mit Vor- und Nachnamen auch machen, aber: Es ist schwierig, weil man nicht weiß ob Person als erstes den Vornamen und den Nachnamen eingibt, deswegen wurde die automatische Übernehmung nur Probleme machen. Ebenso wenn Person den Namen noch nicht vollständig eingegeben hat, dann wird nur diese Hälfte vom Namen übernommen, was auch sinnlos ist. Deswegen habe ich es nur für die Matrikelnummer so gemacht. 

Nach dem Speichern des Schülers wird man automatisch wieder zur Gutschein-Erstelluung zurückgeleitet. Der neu angelegte Schüler ist dann auch automatisch ausgewählt, sodass der Gutschein ohne wieder eine Suche zu machen, sofort erstellt werden kann.
