## Schüler Dropdown Auswahl
Am Anfang habe ich bei der Erstellung eines Gutscheins ein klassisches Dropdown-Menü verwendet, um Schüler auszuwählen. Es wurden alle Schüler/Innen aus der Datenbank geladen und angezeigt. Es hat funktioniert, aber es war unübersichtlich. Deshalb habe ich nun eine Suchfunktion dazuprogrammiert.

### Schülersuche (Autocomplete)
Beim Feld „Schüler:in suchen“ kann man nun Anfangsbuchstabe(n) des Vor- oder Nachnamens eingeben und es kommen passende Ergebnisse. Ebenso kann man anfangen, die Matrikelnummer einzugeben und es erscheinen passende Schüler/innen mit Nummern. Die gewünschte Person kann anschließend direkt angeklickt werden. Weiters kann man auch die Matrikelnummer komplett eingeben und dann ENTER drücken, wodurch der Schüler bzw. die Schülerin direkt ausgewählt wird. Genau gleich funktioniert es auch beim Namen. Dabei ist die Klein- und Großschreibung egal, ebenso die Reihenfolge, also Vorname Nachname oder Nachname Vorname.

Nach der Auswahl wird die Person im Formular angezeigt und intern wird die entsprechende `student_id` gespeichert, damit beim Erstellen des Gutscheins eindeutig der richtige Schüler bzw. die richtige Schülerin verwendet wird.

### Umsetzung der Suche
Für die Suche wurde im `StudentController` eine zusätzliche Methode `search()` erstellt. Diese Methode verarbeitet die Suchanfrage und liefert passende Schüler/innen aus der Datenbank zurück.

```php
public function search(Request $request)
    {
        $query = trim($request->get('q', ''));

        if ($query === '') {
            return response()->json([]);
        }

        $students = Student::query()
            ->where(function ($q) use ($query) {
                $q->where('first_name', 'like', $query.'%')
                    ->orWhere('last_name', 'like', $query.'%')
                    ->orWhere('student_id', 'like', $query.'%')
                    ->orWhereRaw("CONCAT(first_name, ' ', last_name) like ?", [$query.'%'])
                    ->orWhereRaw("CONCAT(last_name, ' ', first_name) like ?", [$query.'%']);
            })
            ->orderBy('last_name')
            ->orderBy('first_name')
            ->limit(10)
            ->get([
                'id',
                'student_id',
                'first_name',
                'last_name',
            ]);

        return response()->json($students);
    }
```
Ebenso kann man die komplette Matrikelnummer eingeben und mit ENTER bestätigen und der Schüler wird ausgewählt. Genau gleich funktioniert es auch beim Namen, da ist die Klein- und Großschreibung, sowie Reihenfolge (Vorname dann Nachname oder andersherum) egal.

## Schülererstellung beim Gutschein erstellen
Es kann vorkommen, dass es einen Schüler im System noch nicht gibt und das merkt man erst bei der Gutscheinerstellung selbst. Deswegen habe ich es so gelöst, dass unter den Suchergebnissen ein Hinweis erscheint, dass keine passende Person gefunden wurde. Zusätzlich wird dann ein Link angezeigt, über den die Person einen neuen Schüler erstellen kann. 

Beim Draufklicken des Links, wird die zuvor eingegebene Matrikelnummer automatisch übernommen. Voraussetzung ist das nur Zahlen eingegeben wurden - dadurch muss man die Nummer nicht erneut eingeben. Ich wollte das mit Vor- und Nachnamen auch machen, aber: Es ist schwierig, weil man nicht weiß ob Person als erstes den Vornamen und den Nachnamen eingibt, deswegen wurde die automatische Übernehmung nur Probleme machen. Ebenso wenn Person den Namen noch nicht vollständig eingegeben hat, dann wird nur diese Hälfte vom Namen übernommen, was auch sinnlos ist. Deswegen habe ich es nur für die Matrikelnummer so gemacht. 

Nach dem Speichern des Schülers wird man automatisch wieder zur Gutschein-Erstelluung zurückgeleitet. Der neu angelegte Schüler ist dann auch automatisch ausgewählt, sodass der Gutschein ohne wieder eine Suche zu machen, sofort erstellt werden kann.

## Mehrere Gutscheine gleichzeitig erstellen (für mehrere Schüler mit gleichen Gutschein-Daten)
Am Anfang konnte beim Erstellen eines Gutscheins nur eine einzelne Schülerin bzw. ein einzelner Schüler ausgewählt werden. In der Praxis kommt es jedoch häufig vor, dass mehrere Schüler/innen für die gleiche Leistung einen Gutschein erhalten sollen (z. B. bei Mithilfe bei Veranstaltungen). Deshalb habe ich diese Funktion erweitert.

### Auswahl mehrerer Schüler/innen
Die Schülersuche funktioniert weiterhin über das Suchfeld mit Autocomplete. Wenn eine Person ausgewählt wird, wird sie nun zu einer Liste der ausgewählten Schüler/innen hinzugefügt. Diese Liste wird direkt unterhalb des Suchfeldes angezeigt. Dort sieht man alle ausgewählten Personen und kann sie auch wieder entfernen.

Intern werden die ausgewählten Schüler/innen im Formular als Array gespeichert `(student_ids[])`. Dadurch können mehrere Schüler-IDs gleichzeitig an den Server übermittelt werden.

### Erstellung der Gutscheine
Im `VoucherController` habe ich die `store()`-Methode angepasst. Statt nur einen Gutschein zu erstellen, wird nun für alle ausgewählten Schüler/innen einer erstellt. Jeder Gutschein wird aber mit den gleichen Basisdaten erstellt (Vorlage, Art, Anzahl und Ablaufdatum). Dadurch können Lehrpersonen mehrere Gutscheine gleichzeitig erstellen, ohne das Formular mehrfach ausfüllen zu müssen.

## Probleme derzeit noch

### Verlust der Auswahl der Schüler beim Anlegen eines neuen 
Wenn bereits mehrere Schüler/innen ausgewählt waren und anschließend eine neue Person angelegt wurde, habe ich folgendes Problem erhalten:
Nach dem Speichern und der Weiterleitung zurück zur Gutschein-Erstellung war nur der neu erstellte Schüler ausgewählt. Die zuvor ausgewählten Schüler/innen waren verloren. Der Grund dafür war, dass die ausgewählten Schüler im Frontend nur im JavaScript gespeichert waren. Beim Wechsel auf die Seite zur Schülererstellung wurde die Seite jedoch neu geladen und diese Information ging verloren.

Um dieses Problem zu lösen, habe ich die bereits ausgewählten `Schüler-IDs` zusätzlich in der URL mitgeschickt (`selected_ids`). Dadurch kann ich die zuvor ausgewählten Schüler beim Zurückkehren zur Gutschein-Erstellung wiederhergestellen. Nach dem Laden werden diese IDs aus der URL gelesen und die entsprechenden Schüler automatisch wieder zur Auswahl hinzugefügt.

### Schüler erscheinen wieder nach dem Entfernen
Ein weiteres Problem trat auf, wenn alle ausgewählten Schüler/innen wieder entfernt wurden. Habe ich getestet, ob ich Formular ohne Schüler abschicken kann. Natürlich trat Validierungsfehler auf, aber irgendwie wurden die zuvor ausgewählten Schüler wieder hereingeladen, weil sie ja in der URL gespeichert waren.

Um dieses Problem zu lösen, habe ich nach dem Wiederherstellen der Schüler die URL automatisch bereinigt. Dadurch werden die Parameter aus der Adresszeile entfernt und die Seite hat wieder eine „saubere“ URL. Somit ist das Problem nun gelöst.

### Doppelte Fehlermeldung
Zusätzlich wurde die Fehlermeldung „Bitte wählen Sie mindestens eine Schülerin oder einen Schüler aus.“ zweimal angezeigt. Einmal ganz oben auf der Seite und einmal direkt unter der Schülerauswahl. Die Ursache war, dass die Seite eine globale Fehlerliste/meldung hatte `($errors->any())` und auch eine feldspezifische Fehlermeldung `(@error('student_ids'))`. Dadurch wurde derselbe Fehler zweimal ausgegeben. Da es für die Benutzer übersichtlicher ist, wenn die Fehlermeldung direkt beim betroffenen Feld angezeigt wird, habe ich die globale Fehlerliste (die immer ganz oben angezeigt worden wäre) entfernt.
